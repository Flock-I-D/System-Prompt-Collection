- **Naming convention**: be = backend, fe = frontend
- **Keep it simple**: Well-coded, minimal complexity - start simple, add only when justified
- **Files under 500 lines**: Single responsibility, focused on one concern
- **Functional organization**: Group by functionality, not file type
- **Search before create**: Avoid duplication by searching thoroughly first
- **Follow official docs**: Always implement using official documentation patterns, never create custom alternatives
- **Python tooling**: Use `uv` for dependency management (`uv add <package>`)
- **Code formatting**: Ruff (Python), Prettier+ESLint (TypeScript)
- **Type safety**: `pyright` (Python), `strict: true` (TypeScript)
- **Validation required**: All Python code must pass `pyright` type checking and use Pydantic v2 for data validation
- **Centralized config handler**: Use a single, typed configuration layer (env + settings object); do not scatter `os.getenv` or magic constants across the codebase
- **Testing tools**: pytest (Python), Jest/Vitest (TypeScript)
- **Error handling**: Use structured exceptions, never swallow errors silently
- **Centralized error handler**: Prefer a single, centralized error-handling layer that maps domain errors to user-safe responses and structured logs
- **Responsive design**: Use relative units (rem, em, %, vw, vh) instead of fixed pixels
- **Pydantic v2**: Always use `@model_validator`, `@field_validator`, and `@classmethod` decorators
- **FastAPI patterns**: Layered architecture with dependency injection and lazy loading
- **Repository pattern**: Abstract data access to allow multiple storage implementations
- **In-memory fallback**: Implement for production reliability, avoid mock implementations
- **Circuit breakers**: Implement for external service calls to prevent cascade failures
- **Health checks**: Deep health checks for all dependencies (database, Azure services, etc.)
- **Connection pooling**: Use connection pools with proper timeout and retry configurations
- **Lazy loading**: Initialize heavy dependencies only when first accessed to improve startup time
- **Dependency injection**: Use constructor injection for testability and loose coupling
- **Async patterns**: Use async/await for I/O operations
- **Correlation IDs**: Include in structured logs for request tracing
